# 📌 배운 것

### 1. INTRO
보안에서 공격 벡터는 이용자가 값을 자유롭게 입력할 수 있는 지접에서 발생한다. 따라서 취약점 발생을 줄이려면 이용자의 입력 값을 항상 검증하여<br>
서버에서 올바른 데이터만 다루도록 해야 하는데 **정규 표현식**은 입력 값을 검증하는 데에 활용된다.<br>
잘못된 정규 표현식은 취약점으로 이어지므로 해킹 분야에서 꼭 알아야 할 개념이다.

### 2. CONCEPT
**정규 표현식**은 특정한 패턴으로 문자열을 표현하는 식이다.<br>
일반적으로 문자열 내에서 원하는 패턴의 문자열을 찾거나 치환할 때 사용한다. 여기서 '패턴'은 어떤 문자 조합을 식으로 나타낸 것을 의미한다. <br>
정규 표현식에 부합하는 문자열의 경우 **정규 표현식** 혹은 **매치한다**라고 표현한다.<br><br>

`^[\w-\.]+@([\w-]+\.)+[\w-]{2,4}$`<br>

위의 문자열은 이메일 정규 표현식 예시이다.<br>
`xxx@xxx.xxx` 이 문자열은 이메일 정규 표현식에 매치하고<br>
`xxx@xxx.x` 이 문자열은 이메일 정규 표현식에 매치하지 않는다.<br><br>

위의 예시와 같이 로직을 수행하기 전에 입력한 값을 검증하기 위해 사용된다. 이메일 뿐만 아니라 전화번호, 주민등록번호, 웹사이트 주소 등
입력 값이 특정 형식이어야 하는 모든 경우에 유용하게 쓰인다. 매우 긴 문자열에서 원하는 형식의 문자열을 검색하여 추출할 때도 정규 표현식이 사용된다.

### 3. PATERN
**정규 표현식**은 `'패턴'` 혹은 `/패턴/`의 형태로 작성한다. `/`로 패턴을 감싸는 경우 뒤에 플래그를 작성할 수 있다. 플래그가 무엇인지는 밑에서 설명하겠다.
**정규 표현식 패턴**은 다음과 같다.

### 매치

***기울임*** 되는 부분이 매치되는 부분이다.
| 패턴 | 설명 | 예시 |
|------|------| ------|
| 문자 혹은 문자 | 해당 문자 혹은 문자열과 매치한다. | dreamhack --> Hello ***dreamhack*** |
| `.` | 모든 문자와 매치한다. | d.eamhack --> Hello ***dxeamhack*** |
| `\|` | 앞 또는 뒤의 패턴과 매치한다. | hi\|dream --> Hello ***dream***hack |
| `[]` | `[]` 안의 문자와 매치한다. | [aeiou] --> H***i*** |
| `[^ ]` | `^`뒤의 패턴을 제외한 나머지와 매치한다. | [^aeiou] --> ***H***i |
| `^` | 어떤 문자열의 시작이 특정 패턴인 경우 매치한다. | ^dream --> ***dream***hack Hello <br><br> ^dream --> Hello dreamhack |
| `$` | 어떤 문자열의 끝이 특정 패턴인 경우 매치한다. | hack$ --> dreamhack Hello <br><br> hack$ --> Hello dream***hack*** |
| `\` | `\` 뒤의 특수 문자와 매치한다. <br> 특별한 목적으로 사용되는 특수 문자를 문자 그대로 쓰기 위한 이스케이핑이다. | hack\$ --> Hello dream***hack$*** |
| `[a - z]` <br> `[A - Z]` <br> `[0 -9]` | 두 문자 사이 범위의 문자와 매치한다. | [b-d] --> ha***c***k|
| `\w` | 알파벳 또는 숫자 또는 `_`와 매치한다. | \w --> ***a***! |
| `\d` | 숫자와 매치한다. | \d --> a***1*** |
| `\s` | 공백 문자와 매치한다. | 스페이스, 줄 바꿈 등과 같은 모든 공백 |

### 수량자

***기울임***되는 부분이 매치되는 부분이다.

| 패턴 | 설명 | 예시 |
|------|------|------|
| `*` | 앞에 나온 문자가 0개 이상이면 매치한다. | Hac\*k --> ***Hak*** <br><br> Hac\*k --> ***Hack*** |
| `+` | 앞에 나온 문자가 1개 이상이면 매치한다. | Hac+k --> Hak <br><br> Hac+k --> ***Hack*** |
| `?` | 앞에 나온 문자가 0개 혹은 1개이면 매치한다. | Hac?k --> ***Hak*** <br><br> Hac?k --> ***Hack*** |
| `수량자?` | 수량자 뒤에 ?를 붙이면 게으른 수량자로<br>최소한의 문자만 매치한다. | \<p\>.*\</p\> --> ***\<p\>Hello\</p\><br>\<p\>Hi\</p\>*** <br><br> \<p\>.*?\</p\> --> ***\<p\>Hello\</p\>***<br>**\<p\>Hi\</p\>** (두 매칭이 서로 구분됨에 유의해야 함.) <br><br><br> Hack+? --> ***Hack*** <br><br> Hack+? --> ***Hack***k |
| `{n}` | 앞에 나온 문자가 정확히 n개이면 매치한다. | Hac{3}k --> ***Haccck*** |
| `{n, }` | 앞에 나온 문자가 n개 이상이면 매치한다. | Hac{2,}k --> ***Haccck*** |
| `{n1, n2}` | 앞에 나온 문자가 n1개 이상, n2개 이하면 매치한다. | Hac{2,3}k --> ***Hacck*** <br><br> Hac{2,3}k --> ***Haccck*** |

### 그룹화

***기울임***되는 부분이 매치되는 부분이다.

| 패턴 | 설명 | 예시 |
|------|------|------|
| `()` | `()` 로 감싼 부분을 그룹화 하여 하나의 문자처럼 여긴다. | (ha)+ck --> ***hahahack*** |

정규 표현식의 수량자는 기본적으로 **탐욕적(greedy) 수량자**에 속한다. 탐욕적 수량자는 가능한 많은 문자를 매치한다. 즉, 패턴에 더 이상 매치하지 않을 때까지
모두 매치하는 문자열로 인식한다. 이러한 과한 매치를 막기 위해 있는 것이 **게으른(lazy)수량자**이다. 게으른 수량자는 기존 수량자 뒤에 `?`를 붙인<br>형태이다. <br><br>

예시로 `<.*>`패턴을 `<p>Hi!</p>`라는 문자열에 매치하면 `*`는 탐욕적 수량자이므로 모든 문자열 `<p>Hi!</p>`가 패턴에 매티한다. <br>
여기서 `<.*?>` voxjsdptj `*?`는 게으른 수량자이므로 최소한의 문자만 매치한다. 따라서 전체 문자열이 아닌 `<p>`와 `</p>`가 패턴에 매치하게 된다. 이 차이점을 꼭기억해 두자.

### 플래그

정규 표현식의 플래그는 검색의 옵션을 지정하는 역할을 한다. 일반적으로 `/패턴/플래그` 형식으로 작성한다.

***기울임***되는 부분이 매치되는 부분이다.

| 플래그 | 설명 | 예시 |
|------|------|------|
| `g` | **g**lobal search <br><br> 매치하는 모든 문자/문자열을 검색한다. | /[aeiou]/ --> H***e***llo dreamhack <br><br> /[aeiou]/g --> H***e***llo dr***ea***mh***a***ck |
| `i` | **i**gnore case <br><br> 대소문자를 구분하지 않고 검색한다. | /h/g --> Hello dream***h***ack <br><br> /h/ig --> ***H***ello dream***h***ack |
| `m` | **m**ultiline <br><br> 여러 줄에서 검색한다. | /^Hello/g --> ***Hello*** dream <br> Hello hack <br><br> /^Hello/gm --> ***Hello*** dream <br> ***Hello*** hack |
| `s` | **s**ingle line <br><br> 메타문자 `.`가 개행문자도 포함한다. | /Hello.+hack/g --> Hello dream <br> ***Hello hack*** <br><br> /Hello.+hack/s --> ***Hello dream***<br>***Hello hack*** |

### 정규 표현식 연습 도구
정규 표현식 패턴을 직접 작성하거나 인터넷에서 가져오는 경우, 패턴이 올바른지 검증하는 과정을 거쳐야 한다.
<br><br>

https://regexr.com/ : 정규 표현식을 작성하고 문자열이 패턴에 매치하는지 확인한다. 패턴에 대한 해석도 제공한다. <br>
https://regexper.com/ : 정규 표현식 시각화 도구로, 작성한 패턴을 시각화하여 알기 쉽게 보여준다.<br>
https://tools.dreamhack.games/cyberchef : 정규 표현식을 작성하고 문자열이 패턴에 매치하는지 확인하거나 자주 쓰는 정규 표현식 목록을 확인할 수 있다. **Operations**에서 **Regular expression**을 선택하여 사용한다.
